# دليل تشغيل تطبيق Hesabên Dikanê (نقطة بيع للحلاقة)

هذا التطبيق: React + Vite + Supabase + Vercel — نفس ستاك Time Capsule تماماً.
اتبع الخطوات بالترتيب. كل خطوة فيها بالضبط وين تكبس.

---

## الخطوة 1 — إنشاء مشروع Supabase

1. روح على https://supabase.com → سجّل دخول → **New Project**.
2. اسم المشروع: `dikan-pos` — اختر Region قريب (مثلاً `eu-west-2` London زي Time Capsule).
3. حط Database Password قوي واحفظه.
4. استنى دقيقتين لين يجهز المشروع.

---

## الخطوة 2 — تشغيل سكربت قاعدة البيانات

1. من القائمة اليسار: **SQL Editor** → **New query**.
2. افتح ملف `schema.sql` (المرفق)، **انسخ كل محتواه**، الصقه في المحرر.
3. اضغط **Run** (أو Ctrl+Enter).
4. لازم تشوف "Success. No rows returned". خلص — كل الجداول والحماية والـtrigger صاروا جاهزين.

> هذا السكربت يعمل تلقائياً: جدول المحل، العمّال، الخدمات، المبيعات، البخشيش، حماية RLS كاملة (كل محل يشوف بياناته بس)، وعند تسجيل أي حساب جديد يُنشأ له محل + 4 خدمات افتراضية.

---

## الخطوة 3 — إعداد تسجيل الدخول (Email)

1. من اليسار: **Authentication** → **Sign In / Providers** (أو Providers).
2. تأكد **Email** مفعّل (Enabled).
3. للتجربة السريعة بدون تأكيد إيميل: روح **Authentication → Sign In / Providers → Email** → أطفئ **Confirm email** مؤقتاً. (شغّله لاحقاً قبل الإطلاق الحقيقي.)

---

## الخطوة 4 — نسخ مفاتيح API

1. من اليسار: **Project Settings** (الترس) → **API**.
2. انسخ قيمتين:
   - **Project URL** (مثل `https://abcd.supabase.co`)
   - **anon public** key (مفتاح طويل)

احتفظ فيهم للخطوة 6.

---

## الخطوة 5 — تجهيز المشروع على جهازك

1. حط كل ملفات مجلد `dikan-pos` في فولدر على جهازك (مثلاً `C:\Users\mahmo\dikan-pos`).
2. افتح Terminal/CMD داخل الفولدر واكتب:

```
npm install
```

(يحمّل React و Supabase و Vite — ياخذ دقيقة.)

---

## الخطوة 6 — وضع المفاتيح

1. في جذر الفولدر، أنشئ ملف اسمه `.env` (انسخ `.env.example` وسمّه `.env`).
2. احطّ فيه القيمتين من الخطوة 4:

```
VITE_SUPABASE_URL=https://YOUR-PROJECT.supabase.co
VITE_SUPABASE_ANON_KEY=YOUR-ANON-KEY
```

3. احفظ الملف.

---

## الخطوة 7 — التشغيل المحلي للتجربة

```
npm run dev
```

افتح الرابط اللي يطلع (عادة http://localhost:5173).
- اضغط **Hesab veke** (إنشاء حساب) → حط إيميلك وباسورد → سجّل.
- بيفتح التطبيق مباشرة (مع 4 خدمات جاهزة). روح **Mîheng** وضيف العمّال.

جرّب: اختر عامل → اكبس خدمات → دفع. شوف الغلّة تتحدّث.

---

## الخطوة 8 — الرفع على GitHub

```
git init
git add .
git commit -m "Dikan POS - initial"
```

أنشئ repo جديد على GitHub (مثلاً `dikan-pos`) واتبع تعليمات الربط:

```
git remote add origin https://github.com/Mahmoud8515/dikan-pos.git
git branch -M main
git push -u origin main
```

> ملف `.gitignore` موجود ويمنع رفع `.env` و `node_modules` (مهم — لا ترفع مفاتيحك).

---

## الخطوة 9 — النشر على Vercel (يشتغل على الآيپاد)

1. روح https://vercel.com → **Add New → Project** → اختر repo `dikan-pos`.
2. Vercel يكتشف Vite تلقائياً. قبل **Deploy**:
3. افتح **Environment Variables** وأضف المتغيّرين (نفس اللي بالـ`.env`):
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_ANON_KEY`
4. اضغط **Deploy**. بعد دقيقة بيعطيك رابط مثل `dikan-pos.vercel.app`.

> ملف `vercel.json` موجود ويصلّح مشكلة الـ404 (SPA rewrites) — نفس اللي عملناه بـTime Capsule.

---

## الخطوة 10 — الاستخدام على الآيپاد

1. افتح Safari على الآيپاد → ادخل رابط Vercel.
2. سجّل دخول بحساب المحل.
3. **Share → Add to Home Screen** — يصير أيقونة زي تطبيق حقيقي، يفتح ملء الشاشة.

---

## ملاحظات مهمة

- **كل محل معزول تماماً**: حماية RLS على مستوى قاعدة البيانات، مش بس بالكود. حتى لو حد حصل على المفتاح العام، ما يقدر يشوف بيانات محل ثاني.
- **العملة**: من Mîheng تختار العملة (دولار/يورو/ليرة/دينار…) وكل المبالغ تتحدّث.
- **اللغة**: زر Kurmancî / Soranî فوق، يحفظ اختيارك.
- **حذف عامل/خدمة**: ما ينحذف فعلياً (يصير `active=false`) — هيك المبيعات القديمة تبقى مربوطة باسمه بالتقارير. ذكي للمحاسبة.
- **النسخ الاحتياطي**: بيانات Supabase محفوظة بالسحابة. للأرشفة الشهرية استخدم زر **Derxe CSV** بصفحة الحسابات.

---

## لو صار خطأ

- **التطبيق فاضي بعد الدخول / "Tê barkirin" ما يوقف**: تأكد إنك شغّلت `schema.sql` كامل (الخطوة 2)، وإن الـtrigger أنشأ المحل. افحص: SQL Editor → `select * from shops;`
- **خطأ بالدخول**: تأكد Email provider مفعّل، ولو فعّلت Confirm email لازم تأكد الإيميل أول.
- **خطأ بالمفاتيح**: تأكد ما في مسافات زيادة بالـ`.env`، وإن المتغيّرات موجودة بـVercel.

بالتوفيق! لما تكون جاهز للتسويق، افتح المحادثة الجديدة بالملخص اللي أخذته.
